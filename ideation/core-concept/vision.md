## Data Architecture

**AI-first requires AI-native data.** We use dual-database architecture:

| Layer | Technology | Purpose |
|-------|------------|---------|
| **Structured** | PostgreSQL | Users, businesses, tasks, relationships |
| **Vector** | pgvector | Semantic search, memory, context matching |
| **Files** | .bos.md | User-editable business context |

**Why both databases?**

Structured DB gives precision: "Find conversations from last month"
Vector DB gives relevance: "Find content about pricing decisions"
Combined: Precise AND relevant results.

**This powers:**
- Memory retrieval (recent + semantically relevant)
- Progressive disclosure (vector similarity → disclosure level)
- Agent routing (query → most relevant agent)
- Unified search (docs, tasks, conversations in one query)

## The Autonomy Progression

A key philosophy: the AI co-founder evolves its relationship with you over time.

### Early Stage — Guided Learning
- Hands-on guidance through every decision
- Encourages users to review and understand frameworks
- Teaches the "why" behind recommendations
- Heavy involvement, frequent check-ins
- Goal: Build foundational knowledge

### Growing Stage — Balanced Partnership
- User leads, AI supports
- Handles routine tasks autonomously
- Intervenes on important decisions
- Asks permission less, acts more
- Goal: Enable confident execution

### Mature Stage — Autonomous Co-Pilot
- Executes independently within boundaries
- Surfaces only what matters
- User sets direction, AI handles details
- Minimal intervention, maximum support
- Goal: Scale the entrepreneur's capabilities

**The goal is not dependence. The goal is to make you a better entrepreneur.**

The AI should make itself less necessary over time—not more.
